## 一、批测暴露的问题:负样本严重假阳性

旧引擎对**明显无健康关联**的政策硬造 8–25 条健康路径(假阳性),例:

| 政策 | 旧版路径数 |
|---|---|
| 04 表彰决策咨询研究成果奖 | 25 |
| 02 无人机适飞空域通告 | 22 |
| 10/16/26/31/33 防空警报试鸣 ×5 | 11–22 |
| 13/14/15/19/29 表彰/授予/慈善奖 | 8–20 |
| 25 基础研究高质量发展 | 33 |
| 28 增值税留抵退税 / 30 国资充实社保 | 8–19 |

**根因(烟枪)**:行动抽取提示词写了具体示例「**新建某设施、新增某类交通、土地用途改变、人口/就业变化**」,当政策无可抽行动时,LLM **把示例原样照抄成"措施"**(04 抽到的 4 条措施正是这几个示例词),再据此凭空造路径;且造出的链条含 PM2.5/噪声等词,被两轨匹配挂上"有据",假阳性还带权威背书。叠加「通常 5–15 条」逼着硬凑、缺"健康关联门槛"。

## 二、修复:三道防线(`hia_screen.py`)

1. **健康关联门槛**:抽取提示词**删示例** + 让模型显式输出 `health_relevant`;非健康(表彰/任免/纪念/财税/程序)→ `analyze` 直接判「关联很弱、0 路径」。
2. **逐行动展开门控**:展开提示词加"逐行动判健康关联,无关行动→0 条";总数 ≤12;批判阶段默认不补(不堆推测性/脆弱群体细分路径)。
3. **安全网 + 主题兜底**:模型把"绿色低碳/生态/能源"误判 false 时,按**标题(文件名)关键词**纠回 true;健康相关但未枚举出行动→取《》标题作主题兜底 + 展开时提示"据文档充分展开、不要返空"(去掉"关于印发…的通知"壳,避免被当成程序性通知)。
