# HIA 引擎重构方案:线性漏斗 → 「干预 × 决定因素」影响矩阵

> 状态:设计稿,待拍板后实现。推倒现有 `funnel.py`(R0–R7)线性管线,按影响矩阵重建。
> 线上仍旧效益版;新引擎离线开发,验过再接 `app_nicegui`、退役旧 `analyze`。

## 0. 为什么推倒

系统要交付的就五个问题,**单位都是「某条干预」**:
A 有无健康危害 / B 通过何种路径 / C 政策有无措施 / D 没有则建议什么措施 / E 可参考哪些标准。

其中 **B/C/D/E 全挂在「决定因素」上**(路径中段=决定因素、措施=控制该因素、标准=该因素的限值)。
所以原子单元不是"一条危害链",而是 **(干预 × 决定因素)** 这个格子——这正是 HIA 实务的**影响矩阵**。

线性漏斗里 **R3 残差召回、R5 语义合并**那一大套,本质是在补偿"没有决定因素当主键"。把决定因素立成脊椎,
**不重 / 不漏 变成结构性保证**,LLM 只需在"列全干预 + 判准有害"两处使劲。

## 1. 交付物 = 影响矩阵(数据模型)

```
HarmCell(一个有害格子):
  intervention: {id, quote(逐字原文), gist(提炼)}   # 哪条干预
  hub:          {id, name}                          # 哪个决定因素(脊椎键)
  why:          str                                 # 为何该干预加剧该因素(锚原文)
  chain:        [节点…]                              # 路径:干预→…→因素→不利结果(B)
  outcome_q:    int                                 # 由 hub.outcomes 查表得
  valid:        bool                                 # 复核(S4)
  mitigation:   已含|不足|未提及                       # 政策有无措施(C)
  mit_quote:    str                                 # 措施原文出处(可审计)
  measures:     str                                 # 建议措施(D)
  standards:    [证据卡…]                            # 参考标准(E,按因素 join)
```
**(intervention_id, hub_id) = 去重主键**:同一干预对同一因素的多条措辞,天然是一格。

## 2. 脊椎:`determinants.py` 怎么用(地基已够,零改动起步)

- **37 个触发性枢纽**(39 减去 `triggering=False` 的 INCOME/EDU)= 筛查清单。每枢纽配**方向正确的危害问法**
  (暴露类问"增加/加剧",好东西类问"减少/削弱";完整 37 问法见附录 A)。
- **`hub.outcomes`** → 该因素落初筛表第几题(AIR→Q2/Q5/Q6…)→ **题号查表得,不再 LLM 判**。
- **`hub.triggering=False`(INCOME/EDU)** → **泛化门**:失业/收入/教育→健康 的结构性连锁不成路径,
  从筛查清单里就不问 → 泛化套链从源头挡掉。
- **`hub.aliases`** → S6 给"参考标准"按因素匹配 `hia_evidence` 卡用(暂关键词,后续可给证据卡补 hub_id 精确 join)。
- ⚠ `templates_harm.json`(80 模板)**作废**(上一轮训练产物),不再 join。

## 3. 新管线 S1–S7(矩阵填充,非闯关)

### S1 干预抽取(LLM·召回)
- **职责**:抽全政策**引入/实质改变**的干预(行动/要素),**不做"是不是干预"的二元 gate**。
- 入:政策原文(≤25k)。出:`[{id, quote, gist}]`。
- **★关键改进**:旧 R0 要下"干预/非干预"布尔,被"规划/管理/指引"措辞带偏 → 漏报(城中村改造)。
  S1 **只列政策做了什么、不下判断**;"有没有害"全交给 S2 矩阵。纯规则政策(上市公司治理)→ S2 命中不了任何
  因素 → 空矩阵 → 0 危害。**于是 R0-gate 漏报这个硬伤结构性消失。**
- 质量:`quote` 逐字锚原文(不臆造);标题/正文强干预词安全网防漏列(穿透制度措辞,扫 新建/改建/扩建/
  拆迁/新设/放松防护/扩产能)。

### S2 危害筛查矩阵(LLM·骨架,= A 题)
- **职责**:每条干预 × 37 枢纽清单逐列问"是否引入/加剧该因素危害?";命中给 `(干预, hub, why)`。
- 入:S1 干预 + 37 危害问法清单。出:矩阵有害格子 `[{intervention_id, hub_id, why}]`,空格丢弃。
- 一条干预可命中多枢纽(新建化工园区→AIR+CHEM+WATER+WASTE);why **必须锚在干预内容**上、不空泛。
- 质量一网打尽:**不漏(因素层)**=固定清单逐列强制覆盖;**不重**=(干预,因素)主键;**泛化**=只问触发性枢纽;
  **无害自然过滤**=防护/纯效益命中不了任何危害问法(矿山汛期"配备备用电源"→不加剧任何因素→0,反转问题消失)。

### S2.5 完整性兜底(LLM·一次,非循环)
- **职责**:重读原文,问"还有没有漏掉的**干预**、或漏掉的**(干预,因素)有害格子**?" 补进矩阵。
- 取代旧 R3 的多轮残差循环——**只过一次**(清单已给了系统性召回,残差只补长尾)。漏判的真危害在此回收。

### S3 路径补全(LLM·受限生成,= B 题)
- **职责**:每个有害格子 → 写因果路径 `干预 → (中间环节) → 决定因素 → 不利健康结果`。
- **决定因素是给定中段**,模型只填两头;**路径长度参数**(默认 4)封顶,防长链臆造。
- outcome_q 由 `hub.outcomes` 候选里挑一个最贴的。

### S4 路径复核(LLM·精度闸,杀)
- **职责**:逐条复核——是否**特定+可预见+锚定原文**?杀掉 S2 过度命中、滑过来的反转/泛泛办砸/跨段。
- 沿用旧 R2 的"特定 vs 泛化 + 固有/特征事故留 + 泛泛办砸杀 + 可被措施避免≠剔"原则,但**更轻**
  (骨架已挡掉大部分);误杀的由 S2.5 那类回收,不在此设残差。

### S5 措施判定(LLM·锚定,= C 题)
- **职责**:每条存活路径 → 政策原文**有无控制该危害的措施**?已含/不足/未提及。
- **判"已含/不足"必须逐字摘录原文那条措施(`mit_quote`);摘不出 = 未提及。** → 可审计,治旧 R6 黑箱。

### S6 建议与标准(LLM + 确定性,= D + E 题)
- **职责**:有缺口的 → ① 生成**建议措施**(结合本政策,口吻随 mitigation);② 按 `hub` 匹配 `hia_evidence`
  挂**参考标准/限值**(D 现场生成=定制;E 标准=客观引用,来源透明)。

### S7 聚合(代码,= 初筛表)
- (干预,因素) 格子 → 按 outcome_q 聚到 10 题;"是/否"由**措施缺口**驱动(已含=已控=否;不足/未提及=是)。

## 4. 质量要求落在哪(结构性 vs LLM 死磕)

| 要求 | 谁保证 | 类型 |
|---|---|---|
| 不漏·干预层 | S1 抽全 + S2.5 完整性 + 强干预词安全网 | LLM 死磕 |
| 不漏·因素层 | **S2 固定 37 清单逐列覆盖** | **结构性** |
| 不重 | **(干预,因素) 主键去重(代码)** | **结构性** |
| 路径合理 | S3 因素给定 + S4 复核 | LLM |
| 不臆造 | S1 锚原文 + S2 锚 why + S4 复核 + S5 锚措施出处 | LLM |
| 不泛化 | **只问触发性枢纽(INCOME/EDU 不问)** | **结构性** |

→ **不重/不漏因素/不泛化 三条变成结构白送**,LLM 只在"列全干预、判准有害、路径锚定、措施摘录"上使劲。

## 6. 实施阶段(增量,每步验,不一把梭)

- **P0 脚手架**:新模块 `matrix.py`(数据模型 + S1–S7 桩);`funnel.py` 保留作参照,验过再退役。

- **P5 接入**:`matrix.run` 接 `app_nicegui` 备选管线;前端/docx 渲 mitigation/measures/standards;退役 `analyze`。

## 7. 开放决策 / 风险

1. **一格多危害**:(化工园区,CHEM) 可能"正常排放"+"事故泄漏"两条 → 允许**每格 ≤2**(正常工况 + 特征事故),仍有界。
2. **因素盖不全的边角危害** → 留 `OTHER` 出口 + 定期回补枢纽(别硬塞错枢纽)。
3. **S1 漏列干预** = 最大风险(后面全空)→ 强干预词安全网 + S2.5 兜;仍需重点验。
4. **S2 假阳**(滥说"加剧空气污染")→ S4 兜,但要调问法严度。
5. **标准(E)精度**:`hia_evidence` 暂关键词匹配;后续给卡补 `hub_id` 改精确 join(守"贴错比没有更糟")。
6. **flash 真方差**(R0-gate 那类已随 gate 取消缓解;计数仍抖)→ 裁质量用稳样多跑或 pro 终判。

## 附录 A:37 触发性枢纽的危害问法(S2 清单)

**〔物质环境 20〕** AIR 加剧大气污染(PM/NOx/SO₂/VOCs)? · NOISE 增加噪声/振动? · WATER 污染水体/损害饮用水? ·
SOIL 造成土壤/重金属污染? · CHEM 增加危化品暴露? · WASTE 增加固废/危废暴露? · WORK_ENV 增加职业危害/工伤? ·
INDOOR_AIR 加剧室内空气污染? · ODOR 产生恶臭/异味? · VECTOR 增加病媒孳生? · RADIATION 增加辐射暴露? ·
HEAT 加剧高温/热暴露? · CLIMATE 增加碳排放/气候风险? · HOUSING 恶化居住条件(潮湿/霉/结构)? ·
CROWDING 加剧居住拥挤/人群聚集? · SANITATION 恶化环境卫生/污水处理? · FOOD 损害食品安全/食物供给? ·
ROAD 增加道路交通事故/伤害? · BUILT_SAFETY 增加建筑/消防/跌倒隐患? · GREEN 减少绿地/开放空间可达?

**〔行为 4〕** TOBACCO 增加烟草暴露? · ALCOHOL 增加有害饮酒? · DIET 恶化膳食(高盐糖/营养不足)? · PA 减少体力活动机会?

**〔社会心理 6〕** STRESS 加剧心理社会应激? · JOB_STRAIN 加剧工作压力/就业不安全? · FINANCIAL_STRESS 加剧财务压力/降低可负担? ·
SOCIAL_SUPPORT 削弱社会支持/加剧排斥隔离? · CONTROL 削弱掌控感/自主性? · VIOLENCE 增加暴力/暴力威胁?

**〔卫生系统 7〕** ACCESS 降低服务可及? · QUALITY 降低服务质量? · HEALTH_INVEST 削减卫生投入? · INSURANCE 削弱医保/增加灾难性支出? ·
EQUITY 扩大健康不公平? · REGIONAL 加剧区域资源失衡? · EMERGENCY 削弱突发公共卫生应对?

**不问(结构性·非触发)**:INCOME(收入就业)、EDU(教育)——经济/教育→健康的连锁是泛化,不作危害路径。
